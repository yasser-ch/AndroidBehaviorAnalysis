Software Architecture

Three-tier decoupled design

High cohesion between signal generation, analysis intelligence, and SOC visualization. Each layer is independently extensible.

Layer 01
Generation
BehaviorTestApp runs on an Android emulator (Pixel 6 Pro, API 37) and generates five distinct behavioral patterns — from normal HTTP traffic to crash storms and request floods. All telemetry is native, relying on standard Android SDK mechanisms.
JavaAndroid SDK API 37OkHttpPixel 6 Pro AVD
Layer 02
Analysis
The Python profiler continuously polls behavioral signals via ADB. The detector applies weighted z-score derivations to identify anomalies, maps findings to MITRE ATT&CK, and orchestrates a local LLM for root-cause generation. A Flask REST API exposes all intelligence.
Python 3.10+Flask 3.1ADBOllama phi3Z-Score
Layer 03
Visualization
A responsive single-page SOC dashboard polls the backend every second. Five views provide complete operational coverage: live metrics, raw signal feed, MITRE kill chain with IOCs, AI root-cause analysis, and exportable audit reports.
HTML5 / JSChart.jsREST/JSONServer-Sent Events

Detection Engine

MITRE ATT&CK mapped rules

Six detection rules cover the most critical Android runtime attack patterns, each classified with MITRE tactic, technique, and confidence level.

Data Exfiltration Pattern
T1048 · Exfiltration
Detects anomalous outbound data transfer patterns consistent with unauthorized data exfiltration attempts from the device.
CriticalConfidence: High
Request Spam
T1498 · Impact
6 parallel threads with no throttle generate high-volume network floods. Detected via RPM spike and error rate correlation.
HighConfidence: Low
Infinite Retry Loop
T1071.001 · C2
Continuous retry on 401 without exponential backoff, signaling potential C2 communication or broken authentication logic.
HighConfidence: Low
Crash Storm
T1499.004 · Impact
NullPointerException every 500ms creates a cyclic crash pattern. Monitored via per-second crash counter deviation.
HighConfidence: Low
Silent App Hang
T1499 · Impact
Application becomes unresponsive without generating explicit errors. Detected through latency spikes with near-zero RPM.
HighConfidence: Medium
Timeout / Slow Response
T1499 · Impact
10s server delay vs 5s client timeout generates persistent timeout patterns, potentially masking network-layer interference.
MediumConfidence: Low

Getting Started

Up and running in minutes

No complex infrastructure required. Just Python, ADB, and an Android device or emulator.

01
Clone the repository
Fork or clone the project to your local machine.
git clone https://github.com/Abdeljalil-Fajri/AndroidBehaviorAnalysis.git
02
Install Python dependencies
Navigate to the profiler directory and install requirements.
cd profiler && pip install flask flask-cors colorama requests
03
Pull the local LLM (Ollama)
Download and run phi3 locally via Ollama — no API key, no cloud.
ollama pull phi3 && ollama serve
04
Connect your Android device
Enable USB debugging on your device or start an emulator, then verify ADB can see it.
adb devices
05
Launch the profiler
Start the Flask backend. Then open profiler/static/index.html in your browser for the live SOC dashboard.
python main.py

REST API

OpenAPI 3.0 documented

All profiler data is exposed through a Swagger-documented Flask API, ready for SIEM integration and CI/CD pipelines.

MethodEndpointDescription
GET/api/statusServer health, tracking metadata, baseline status
GET/api/signals/currentLatest 1-second signal snapshot
GET/api/signals/historyRolling 60-second telemetry history
GET/api/logsLast 50 raw logcat lines
GET/api/incidentsAll detected incidents for the active session
POST/api/analyzeTrigger on-demand AI root cause analysis via Ollama
POST/api/incidents/:id/triageSet status: acknowledged / mitigated / false_positive
